Theme foundation

This page exercises the custom wiki.css selectors using a representative Wiki.js-like structure. It is a local validation aid, not a Wiki.js runtime component.

Links should use a calm Atlassian-like blue and become underlined on hover or focus. For example, read the reference documentation.

Navigation and page structure

The shell should feel like a compact enterprise documentation workspace.

  • White app bar with subtle border and restrained shadow.
  • Left navigation with pale active state and clear hierarchy.
  • Readable article width for long-form technical content.
  1. Copy the CSS into the Wiki.js custom CSS override area.
  2. Review navigation, article content, and technical blocks.
  3. Keep this preview as a local validation aid only.

Table treatment

Area Selector family Expected result
Navigation .v-navigation-drawer Light surface, subtle active blue, readable labels.
Article .v-main .contents Sentence-case headings and comfortable line height.
Controls .v-btn / .v-speed-dial Blue primary actions, neutral secondary actions.

Code and callouts

Inline code such as python -m http.server 4173 should remain quiet.

/* Wiki.js custom CSS override */
#root .v-application .v-main .contents h2 {
  border-bottom: 1px solid var(--wiki-border-subtle);
}

Info callout: use soft blue fill and a clear left border.

Warning callout: keep the tone visible without becoming loud.

Danger callout: reserve red for destructive or high-risk information.

Success callout: use green as confirmation, not as the main brand color.
